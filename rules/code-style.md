# Code Style

Règles de style applicables au projet :

- Utiliser des noms de variables explicites en `snake_case` pour Python.
- Respecter l’indentation de 4 espaces.
- Préférer `pandas` pour le traitement des données.
- Ajouter des commentaires si la logique métier est complexe.
- Créer des tests pour les fonctions critiques.
